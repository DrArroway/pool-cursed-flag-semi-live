pool-cursed-flag-semi-live

It's easier to change the flag than fix the Reflecting Pool.


Why semi-live?

Fully live video would be nice, sure. But after running the numbers, most people only need the image to feel "recent enough." So we grab a few frames per day/wwek, push them here, and call it semi-live.

Saves on bandwidth, keeps the server happy, and nobody the difference between 8 seconds ago and 6 hours ago anyway.


How it works?

capture.js scrapes the pool cam when I'm connected and if the srcipt don't bug out

Images land in /semiLivePics ang get archived for every one to see 

matrix-tracker.js pulls the average color matrix from the pool area and slaps it onto the flag star backgroung in a 50 area dodgy rectangle with kind of matrix for every stars to get it's own shap of swamp

Calibration sliders just in case or the clouds ascrewing up the swamp again

Real pool colors (not just procedural green sludge)

Cloud compensation (yes, really, but probybly not as good as it should be)

Archive browser so you can watch the bloom evolve day by day

Adjustable extraction algeas j, just because we want the flag grenn (you can turn it off for real data)
